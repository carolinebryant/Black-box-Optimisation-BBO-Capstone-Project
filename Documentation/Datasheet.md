Black-Box Optimisation Capstone Dataset

## Motivation

This dataset was provided by the Imperial College London Computing Department and Emeritus Business School for the capstone project of the Artificial Intelligence and Machine Learning professional certificate. It supports a Black-Box Optimisation challenge in which Bayesian Optimisation is applied to 8 unknown functions of varying dimensionality (2D to 8D), each representing a different real-world scenario (e.g. radiation source detection, drug compound mixing, chemical yield, ML hyperparameter tuning). Creation and maintenance are funded by the ICL/Emeritus professional certificate programme; no external grant or third-party sponsor is involved.

## Composition

The dataset consists of initial observations for 8 black-box functions, each stored in a dedicated folder (`function_1` through `function_8`) containing two NumPy files: `initial_inputs.npy` (the query points) and `initial_outputs.npy` (the corresponding function evaluations). Input dimensionality ranges from 2D to 8D and sample sizes from 10 to 40 points per function. Inputs fall roughly within [0, 1], though some values exceed 1.0. Weekly queries are submitted to the evaluation server as hyphen-separated strings of the form `x_1-x_2-...-x_n`, where each `x_i` is given to six decimal places (e.g. `0.123456-0.654321` for a 2D function). The data is deliberately sparse, providing only a rough initial sketch of each function's landscape, with no ground-truth labels, analytic form, or noise characterisation disclosed. The files represent the full starter data released to students. Train/test/validation splits do not apply, since all points are used to fit the surrogate models and evaluation is performed externally via weekly query submissions. The dataset contains no personal data, no human subjects, no demographic subpopulations, and no sensitive or offensive content.

## Collection process

The initial data points in the `.npy` files were pre-generated by ICL, with the initial data being intentionally sparse as part of the challenge. Beyond this starter set, queries were generated weekly over a 13-week period: each week, one new input point per function was submitted to ICL's evaluation server, which returned the corresponding output. Query selection was driven by surrogate models (Gaussian Processes, Random Forests, Extra Trees, Support Vector Regression, polynomial regression, and a dropout MLP for Function 8) combined with acquisition functions (Upper Confidence Bound, Expected Improvement, and Thompson sampling), tuned per function based on dimensionality, noise behaviour, and observed landscape structure. Because the data is fully synthetic and involves no human or animal subjects, no ethical review, informed consent, or impact assessment was required.

## Preprocessing and uses

Several transformations were applied during modelling: log-abs transforms on Function 1 outputs to handle extreme dynamic range, `log1p` on Function 5 yields, standard scaling of inputs and outputs for the Function 8 MLP, and rank transforms where raw output scales were unstable. The raw `.npy` files are preserved unchanged; all transforms are applied in-memory during modelling and do not overwrite the source data. Candidate points for acquisition were drawn from a mix of strategies depending on the function, including uniform random sampling, Latin Hypercube sampling, targeted perturbations around the best observed point, and dimension-specific sweeps; in some cases, an SVR surrogate was used to shortlist candidates before the main acquisition step. 

**Intended use:** educational demonstration of Bayesian Optimisation under a limited evaluation budget for the Captone project. 

**Inappropriate uses:** the data is too sparse and the underlying functions too domain-abstracted to draw real scientific conclusions about radiation detection, drug dosing, chemical yield, or any of the named application scenarios. Because the functions are synthetic and the scenarios are illustrative only, there are no fairness, representational, or demographic-bias concerns associated with the dataset itself.

## Distribution and maintenance

The dataset is distributed privately to enrolled students of the Imperial College London / Emeritus AI & ML professional certificate and is not publicly available. It is made available through the programme's online learning platform at the start of the capstone module. Terms of use are governed by the programme's enrolment agreement and restrict redistribution. No separate licence is attached to the `.npy` files. The dataset is maintained by the Imperial College London Computing Department in partnership with Emeritus Business School, which also hosts the evaluation endpoint used to score weekly queries. The starter files remain static for the duration of the capstone.
